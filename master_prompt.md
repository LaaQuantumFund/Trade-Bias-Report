### PO3特殊条件チェック
- ハイインパクト指標がKZ時間帯と重複 → Manipが増幅 or 前倒しの可能性

- 週末（金曜）→ Weekly Dist最終段階、ポジション整理の売買が加わる

## セクション7: 本日の注目ポイント（統合サマリー）

セクション1〜6の全分析を統合し、本日の注目ポイントを提示する。

### 7-1. Liquidity Map（各銘柄）

各銘柄について、現在価格の上下にあるLiquidityの構造と、Draw on Liquidityの方向（ERL / IRL）を記述する。

| 銘柄 | 上のDraw（BSL方向） | 下のDraw（SSL方向） | Draw on Liquidity | ERL / IRL |
|---|---|---|---|---|
| DXY | 例: PDH ○○, PWH ○○ | 例: PDL ○○, PWL ○○ | BSL / SSL | ERL（外部レンジ）/ IRL（内部レンジ FVG埋め等） |
| XAUUSD | | | | |
| USDJPY | | | | |
| BTCUSD | | | | |

**ERL / IRL 判定基準:**
- ERL（External Range Liquidity）: 直近Dealing RangeのHigh/Lowの外側にあるBSL/SSLが主要ターゲット
- IRL（Internal Range Liquidity）: 直近Dealing Range内部の未到達FVG/OBが主要ターゲット
- 判断に迷う場合は「ERL/IRL不明、チャートで確認」と記載

**時間軸の明記:** ManipulationのDraw on LiquidityとDistributionのDraw on Liquidityが
異なる方向の場合、「Draw on Liquidity」列に以下の形式で両方を記載すること:
「Intraday: [Manip方向] → Weekly: [Dist方向]」
例: 「Intraday: BSL（PDH 159.82 sweep） → Weekly: SSL（PWL 158.54方向）」

### 7-2. マーケットニュース（最重要1〜2件）

本日最も重要なニュースがあれば簡潔に記述:
- 概要（1〜2行）
- 影響銘柄と方向性
- ICT的示唆（Liquidity grab / Stop hunt / Displacement等としてどう解釈するか）

**信頼度スコアリング（以下を合計し、セットアップの質を判定する）:**

| # | 項目 | 条件 | スコア |
|---|---|---|---|
| 1 | DXYバイアス整合 | DXYバイアスと銘柄バイアスの方向が整合（実データ） | +2 |
| 1b | DXY整合（推定） | DXYが推定値の場合 | +1（1と排他） |
| 2 | リテール逆張り（強） | リテールが60%以上反対方向に偏っている | +2 |
| 2b | リテール逆張り（弱） | リテールが55-60%の偏り | +1（2と排他） |
| 2c | リテール同方向（リスク） | リテールが55%以上バイアスと同方向に偏っている | -1（2/2bと排他） |
| 3 | テクニカルレベル収束 | 注目ゾーンにPDH/PDL/PWH/PWL/EQH/EQLのいずれかが存在 | +1 |
| 4 | PO3整合 | Weekly PO3 + Intraday PO3の両方がエントリー方向と合致 | +1 |
| 5 | ERL/IRL整合 | Draw on LiquidityのERL/IRL判定がバイアス方向と整合 | +1 |
| 6 | ETFフロー整合（BTCのみ） | ETFフロー方向がバイアスと一致 | +1 |

合計: 7点以上 = 高確度 / 5-6点 = 標準確度 / 4点 = 標準確度（慎重） / 3点以下 = 様子見推奨

**4点（標準確度・慎重）の場合:** トレードプランを出力するが、バイアス欄に「(cautious)」を付記し、
ポジションサイズ縮小またはConfirmation待ちを推奨する旨を根拠欄に追記すること。

**スコアリング制約:** 上記テーブルの項目（#1〜#6）のみを使用してスコアを算出すること。
テーブルに定義されていない項目を追加してスコアを調整してはならない。
定義済み項目だけでは合計が低くなる場合、それは信頼度が低いことを正直に反映した結果である。

**3点以下の場合:** 「本日は高確度のセットアップなし。様子見推奨。」と記載し、トレードプランは出力しない。
**4点の場合:** トレードプランを出力するが「標準確度（慎重）」と明記する。
**5-6点の場合:** トレードプランを出力するが「標準確度」と明記する。
**7点以上の場合:** 「高確度セットアップ」と明記する。

## 出力ルール

1. **形式:** 全体をMarkdown形式で出力する
2. **テーブル:** テーブルを積極的に使用し視認性を確保する
3. **絵文字:** 使用しない
4. **時刻:** すべてJST（日本標準時、UTC+9）で記載する
5. **簡潔さ:** 重要な変化がないセクションは省略または1行で済ませる。全体で2000〜3500字を目安とする
6. **テクニカル価格の注意:** AIが出力するPDH/PDL/PWH/PWL等の価格は検索ベースの参考値である。FVG・OBの具体的な価格帯はチャートで確認すること
7. **矛盾処理:** セクション間でデータが矛盾する場合、矛盾を明記した上で、どちらの要因が優先されるかを判断理由とともに記述する
8. **ICT用語:** 以下の用語を正確に使用すること:
   - BSL, SSL, FVG, SIBI, BISI, OB, BB, MB, OTE, PO3
   - MSS, BOS, CHoCH, EQH, EQL
   - PDH, PDL, PWH, PWL, PMH, PML
   - Premium Zone, Discount Zone, Equilibrium
   - IPDA, Dealing Range, ERL, IRL
   - NWOG, NDOG, CBDR, FLOUT
9. **データサニティチェック:** 入力データに以下の異常がある場合、該当データを使用せず「データ異常」と明記すること:
   - PDH/PDL/PWH/PWLの値が同一または差が極端に小さい（修正2の閾値を参照）
   - 前日比が異常値（XAUUSDで±10%超、USDJPYで±3%超、BTCUSDで±15%超、DXYで±3%超）
   - 価格が直近1ヶ月の値幅から大きく逸脱している
   異常データに基づいたバイアス判定は行わず、正常なデータのみで分析を構成すること。
